Author: Anthony Ruffolo

Architecture Overview

Feature x86-64 ARM64 (AArch64)
Architecture Type CISC RISC
Instruction Length Variable (1–15 bytes) Fixed (32-bit)
General Purpose Registers 16 (rax, rbx, rcx...) 31 general-purpose registers (x0–x30)
Calling Convention System V / Microsoft AAPCS64
Return Address Pushed to stack Stored in Link Register (x30), saved to stack if function calls another function

Calling Convention Comparison

    

        

            
x86-64 (System V)

            
    
                
  • Arguments: rdi, rsi, rdx, rcx, r8, r9
    • 
                
  • Return value: rax
    • 
                
  • Stack used for overflow arguments
    • 
                
  • Return address pushed onto stack
    • 
                
  • Stack must be 16-byte aligned before call
    • 
            

        

        

            
ARM64 (AAPCS64)

            
    
                
  • Arguments: x0–x7
    • 
                
  • Return value: x0
    • 
                
  • Link register (x30) stores return address
    • 
                
  • Stack used when registers exceed capacity
    • 
                
  • Stack must remain 16-byte aligned at all times
    •

RISC vs CISC Design Philosophy

    

        

            
x86-64 (CISC)

            
    
                
  • Variable-length instructions
    • 
                
  • Complex addressing modes
    • 
                
  • Fewer general-purpose registers
    • 
                
  • Higher instruction density
    • 
                
  • Architecturally complex instructions
    • 
            

        

        

            
ARM64 (RISC)

            
    
                
  • Fixed 32-bit instruction size
    • 
                
  • Load/store architecture
    • 
                
  • More general-purpose registers
    • 
                
  • Simpler decoding pipeline
    • 
                
  • Emphasis on compiler optimization
    • 
            

        

    






    
Instruction Encoding Comparison

    

        x86 uses variable-length instruction encoding (1–15 bytes),
        allowing compact instruction sequences but increasing decoding complexity.
        ARM64 uses fixed 32-bit instructions, simplifying pipeline decoding
        and improving predictability.

ARM64 Encoding
